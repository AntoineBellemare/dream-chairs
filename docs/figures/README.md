# Figures

Paper-ready figures generated by `python scripts/make_figures.py`. Each is provided as a
**vector PDF** (for LaTeX/print) and a **300 dpi PNG** (for preview/slides). Source data:
the `outputs/dataset_pointe` results and `outputs/chairs`.

Suggested captions:

- **fig1_pipeline** — *System overview.* A dream report is turned into a 3D form
  (Point-E / Shap-E), embedded by OpenShape into a space shared with text, then analyzed
  three ways — clustered into families of shapes, characterized by design-descriptor
  families, and projected onto bodily-posture axes that map to chair parameters — before a
  posture-matched chair is generated. Dashed path: posture can also be read directly from
  the dream text.

- **fig2_corpus_gallery** — *Generated dream-form corpus.* 50 of 100 forms produced by
  Point-E from dream-derived prompts (point colors aid depth reading).

- **fig3_cluster_tsne** — *Families of dream-forms.* t-SNE of the OpenShape embeddings of
  100 forms; KMeans with k chosen by silhouette (k = 9). Insets show each cluster's nearest
  exemplar; legend labels give the top distinctive descriptor and cluster size.

- **fig4_cluster_signatures** — *Cluster characterization.* For each cluster, the
  design-family qualities with the largest contrastive lift over the corpus average — i.e.
  what makes each family of forms distinctive rather than merely typical.

- **fig5_posture_map** — *Posture map.* Each form projected onto two bodily axes
  (orientation: reclined↔upright; enclosure: exposed↔cocooned), colored by shape cluster.
  Forms from the same shape cluster spread across the posture plane, showing posture is a
  distinct, body-relevant coordinate from shape similarity.

- **fig6_dream_to_chair** — *Dream → chair.* For two dreams, the target posture read from
  the text (left) and the chair selected as the best posture match among K generated
  candidates (right).

- **fig7_pointe_vs_shape** — *Generator comparison.* The same five dreams rendered by
  Point-E (points) and Shap-E (surface-sampled mesh); Shap-E yields denser, more solid
  (fabricable) geometry.

- **fig8_mesh_gallery** — *Shap-E mesh corpus.* 24 Shap-E meshes rendered by dense surface
  sampling.

- **fig9_alignment** — *Dream ↔ form faithfulness.* Left: mean text→form alignment per axis
  set (symbols > design families > posture), each well above its shuffled baseline. Right:
  which posture axes survive translation (orientation/tone/weight/enclosure preserved;
  spatial extent and stability largely lost).

- **fig3/4/5_*_clean** — the same cluster map, design-family signatures, and posture map
  recomputed on the 78 forms left after removing the flagged degenerate blobs
  (`scripts/flag_blobs.py` → `clean_ids.json`). Families separate more and their signatures
  are sharper than the full-corpus versions.

All forms are rendered with `dream_chairs/render3d.py` (matplotlib-3D, coolwarm-by-height,
bounding-box cube). Regenerate:

```bash
python scripts/make_figures.py            # full corpus
python scripts/make_figures.py --clean    # clean (blob-free) fig3/4/5
```
